# LLM 모델 비교·선정 보고서

## 1. 비교 목적

본 비교의 목적은 CANIDAY 운영 회의록을 입력했을 때, 회의 요약, 결정사항, Action Items, 리스크, 확인 필요 사항을 가장 정확하게 정리하는 LLM을 선정하는 것이다.

회의록 요약 과업에서는 회의 메모에 없는 담당자, 기한, 수치, 정책을 임의로 생성하지 않는 것이 중요하다. 따라서 단순히 문장을 자연스럽게 작성하는 능력보다, 확정사항과 미확정사항을 구분하고 근거 부족 항목을 명확히 표시하는 능력을 중점적으로 평가했다.

세 모델 모두 동일한 회의록 요약 프롬프트를 입력했다.

- 공통 프롬프트 원문: [evidence/prompt.md](./evidence/prompt.md)
- ChatGPT 결과: [evidence/chatgpt_result.md](./evidence/chatgpt_result.md)

- Gemini 결과: [evidence/gemini_result.md](./evidence/gemini_result.md)

비교 대상 과업은 다음과 같다.

- 과업명: CANIDAY 운영 회의록 요약 및 Action Items 추출
- 입력 데이터: CANIDAY 앱 출시 이후 운영 회의 메모
- 출력 요구사항: 회의 요약, 결정사항, Action Items, 리스크, 확인 필요 사항, 최종 공유용 요약
- 핵심 제한 조건: 회의 메모에 없는 담당자, 기한, 수치, 정책을 지어내지 말 것

## 6. 모델별 결과 요약

### ChatGPT 5.0

ChatGPT는 회의 요약과 결정사항을 비교적 정확하게 정리했다. 특히 SNS 운영안이 검토 단계라는 점과 개인사업자/법인사업자가 미확정이라는 점을 구분했다.

다만 제출된 결과 기준으로 Action Items 표가 중간에 끊겨 있어 전체 형식 준수 여부를 완전히 판단하기 어렵다. 또한 Claude에 비해 리스크와 확인 필요 사항이 구조적으로 충분히 정리되지 않았다.

### Claude Sonnet 4.6

Claude는 세 모델 중 가장 안정적으로 요구 형식을 준수했다. 확정된 결정사항과 제외해야 할 미확정 항목을 별도로 구분했고, Action Items에서도 담당자와 기한이 없는 항목을 임의로 작성하지 않고 미확정으로 표시했다.

리스크와 확인 필요 사항도 담당자, 기한, 정책, 수치로 분류해 실무에서 바로 사용할 수 있는 형태로 정리했다. 회의 메모에 없는 수치나 담당자를 만들어내지 않았기 때문에 환각 방지 측면에서도 가장 우수했다.

### Gemini 3.1 Pro

Gemini는 전체적으로 요청한 형식을 잘 따랐고, 요약과 Action Items도 실무적으로 정리했다. 그러나 답변 앞뒤에 “완벽하게 정리”, “튜터의 피드백 코멘트”처럼 사용자가 요구하지 않은 부가 설명이 포함되었다.

또한 “온보딩 화면을 개선하기로 했다”처럼 원문의 “개선안을 정리하기로 했다”보다 다소 강하게 표현한 부분이 있어, 확정사항을 엄격히 다루는 과업에서는 주의가 필요하다.

## 7. 최종 선정 결론

최종 선정 모델은 Claude Sonnet 4.6이다.

Claude Sonnet 4.6은 결정사항, 미확정 항목, Action Items, 리스크, 확인 필요 사항을 가장 명확하게 분리했다. 특히 회의 메모에 없는 담당자와 기한을 임의로 생성하지 않고 “미확정” 또는 “확인 필요”로 표시해 환각 방지 기준을 잘 충족했다.

또한 Action Items 표의 완성도가 높고, 사내 공유용 요약도 실무에서 바로 사용할 수 있는 수준이었다. 따라서 CANIDAY 운영 회의록 요약 및 Action Items 추출 자동화 과업에는 Claude Sonnet 4.6이 가장 적합하다고 판단했다.
